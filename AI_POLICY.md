# AI Policy

**Version:** 2026-08-01

## Scope

This policy covers the use of artificial intelligence tools in contributions to this project. "AI" refers to:

- **LLM chat interfaces** (e.g., ChatGPT, Claude, Gemini)
- **Full-line code completion** (e.g., GitHub Copilot, Codeium, Tabnine)

## Project AI Usage

For up to and including version 1.0.0, AI was used to scaffold code, write boilerplate, assist with diagnosing issues, advise on architecture and code quality, and write tests. The majority of code was hand-written.

## Contributor Requirements

### Disclosure

You may use AI to contribute code, assist in reporting issues, or perform code reviews. **You must declare AI usage.** Disclosure opportunities are provided in the issue and pull
request templates.

### Issues

If you used AI to write or assist in writing an issue, you must fully understand its content and be prepared to discuss it. Except under extenuating circumstances, if you are
unable to do so, your issue will be closed.

### Pull Requests

If you used AI to write code, you must review every line of AI-generated code and be prepared to explain it. Except under extenuating circumstances, if you are unable to explain
your code, your pull request will be closed.

This includes AI-generated tests. Passing tests do not prove correctness. AI-generated test suites require the same line-by-line scrutiny as any other AI-generated code.

### Code Reviews

If you use AI to assist in reviewing code, you must disclose this and verify the accuracy of every comment, suggestion, and approval.

### Commit Messages and Documentation

AI-generated or AI-assisted commit messages and documentation become permanent project history. Contributors must verify the accuracy of all commit messages and documentation,
regardless of how they were produced.

### Licensing and Intellectual Property

Contributors must affirm that AI-generated code does not knowingly introduce copyrighted or license-incompatible material. You are responsible for the provenance of all code you
submit.

## Policy Versioning

This policy may evolve over time. The version of this policy on the target branch at the time of contribution applies to that contribution.
