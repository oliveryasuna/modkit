# Modkit example mod

A real, working mod built with the Modkit plugin suite — one codebase compiled for **Fabric and NeoForge** across **two Minecraft versions** (a 4-node matrix), exercising nine Modkit plugins: `core`, `loaders`, `metadata`, `mixins`, `dependencies`, `datagen`, `run`, `testing`, and `multiversion`.

## What it does (visible proof it loaded)

- **A mixin** into `MinecraftServer#loadLevel` logs a line when a world loads — the same `@Mixin` class on every loader and version.
- **A registered item** (`modkit_example:example_item`) appears in the creative inventory, with a model and name generated by **datagen**.
- **A JUnit test** (`./gradlew test`) asserts the mod's basics without launching the game.

## The matrix

| Minecraft | Fabric | NeoForge |
|---|---|---|
| 1.21.1 | ✅ | ✅ |
| 1.20.6 | ✅ | ✅ |

Each `(version, loader)` is a Stonecutter node (`versions/<mc>-<loader>`). The only version-specific source branch is a couple of preprocessor lines; the loader-specific code (the Fabric `ModInitializer` vs the NeoForge `@Mod` bootstrap) is branched with Stonecutter `//? if fabric`.

## Building

Modkit is not published to the Gradle Plugin Portal yet, so publish it to your local Maven repository first, then build the example:

```sh
# 1. From the repository root — publish Modkit 0.2.0 to ~/.m2
cd ../..
./gradlew publishToMavenLocal

# 2. Back in this directory — build every node
cd examples/example-mod
./gradlew build
```

`build` compiles, tests, and produces a loadable jar for the **active** node (set in `stonecutter.gradle.kts`). To build a specific node:

```sh
./gradlew :1.21.1-fabric:build
./gradlew :1.20.6-neoforge:build
```

Jars land in `versions/<node>/build/libs/`.

## Running it in-game

Launch the client for a node to see the item and the mixin log:

```sh
./gradlew :1.21.1-fabric:runClient
./gradlew :1.21.1-neoforge:runClient
```

Open the creative inventory to find **Example Item**, and watch the log for the `[modkit_example] … mixin active!` line when a world loads.

## Regenerating assets

The item's model and language file are produced by data generation:

```sh
./gradlew :1.21.1-fabric:runDatagen   # Fabric
./gradlew :1.21.1-neoforge:runData    # NeoForge
```

## How it's wired

- `settings.gradle.kts` — resolves Modkit from `mavenLocal()`, applies the `multiversion` settings plugin, and declares the matrix with `modkitVersions { root { version(...) } }`.
- `stonecutter.gradle.kts` — the Stonecutter controller (active node + the `fabric`/`neoforge` preprocessor constants).
- `build.gradle.kts` — the single central build script shared by every node: applies the Modkit plugins and describes the mod in one `modkit { }` block, choosing the loader/API versions from `stonecutter.current.version`.
